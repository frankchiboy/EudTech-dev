# Netlify 部署全白畫面（白屏）問題解決說明

## 問題描述

網站部署到 Netlify 後，部分用戶遇到「全白畫面」或資源載入失敗，通常是因為 Service Worker 快取機制或瀏覽器快取導致載入到舊版資源、資源損壞，或新版資源與快取不一致。

## 解決方案

### 1. Service Worker 快取徹底清除
- **index.html** 於 <head> 加入自動 unregister 所有舊 Service Worker 腳本，確保每次載入都移除舊快取：
  ```js
  if ('serviceWorker' in navigator) {
    navigator.serviceWorker.getRegistrations().then(function(registrations) {
      for(let registration of registrations) {
        registration.unregister();
      }
    });
  }
  ```
- **public/sw.js** 新版 Service Worker：
  - 安裝時自動 `skipWaiting()`，確保新 SW 立即啟用。
  - activate 時自動清除所有舊快取。
  - fetch 事件直接走網路，不做快取，避免資源殘留。

### 2. 新版本自動偵測與提示
- **index.html** <head> 加入 `<meta name="version" content="__BUILD_VERSION__">`。
- build 後用腳本自動注入版本號（scripts/inject-version.cjs）。
- 全域引入 VersionChecker 組件，偵測到新版本時自動提示用戶刷新頁面。

### 3. Netlify 快取設定
- **public/_headers** 設定：
  - `index.html` 設為 `no-cache`，每次都載入最新版本。
  - 靜態資源（assets）設長效快取，提升效能。

### 4. 其他細節
- **vite.config.ts** base 設為 `/`，確保資源路徑正確。
- 圖片載入體驗最佳化，避免因圖片載入失敗導致白屏。

## 操作步驟
1. 完成上述程式碼與設定修改。
2. 重新部署網站到 Netlify。
3. 通知所有用戶「請重新整理一次頁面」，以確保新版 Service Worker 與快取清除機制生效。

## 成效
- 用戶不再遇到全白畫面或資源損壞。
- 新版本能自動偵測並提示用戶刷新。
- 快取機制安全、穩定。

如需 UX 提示「已自動清除快取，請重新整理」，可再補充。
